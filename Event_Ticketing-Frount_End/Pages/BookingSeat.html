Entity Relationships
🔗

Parent · Booking

Each BookingSeat row belongs to one Booking via bookingId. A single booking can own multiple rows — one per selected seat.

Parent · Seat

Each row references one Seat via seatId. Once assigned, the Seat status is set to Booked to prevent conflicts.

This is a junction / mapping table that resolves the many-to-many relationship between Bookings and Seats. It records the exact timestamp of seat assignment and enables temporary seat locking during the payment window.
